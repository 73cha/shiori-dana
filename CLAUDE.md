## 要件定義
以下の要件を満たしたアプリを開発します。

- NotionのDBでブックーマークを管理
- ブックマークの削除ができる（論理削除: Notionの`published`チェックボックスで管理）
- ブックマークのタイトル・タグの編集ができる（`<dialog>`による編集フォーム）
- ブックマークのリアルタイム検索

## 必要なUI
- リアルタイム検索の検索バー
- ページネーション
- ブックマークのカードUI

## 技術スタック
- フレームワーク: Svelte
- 言語: TypeScript
- ビルドツール: Vite
- アーキテクチャ: SPA
- データフロー: Notion DB → JSON → filterで絞り込み

## 利用したいサービス
- DB: Notion
- リアルタイム検索: Array.filter()による部分一致検索
- ホスティング: Vercel
- ブックマーク更新: Notion API
- 更新通知: トースト表示（async/await）

### ブックマーク更新の流れ

Notion APIのトークンをクライアントに露出させないため、Vercel Serverless Functions（API Routes）を中間に挟みます。

```
UIの更新ボタン
  ↓ POST
/api/v1/bookmarks/update/
  ↓ Serverless Function内で Notion DB → JSON 変換
  ↓ レスポンスとしてJSONを返却
クライアント
  ↓ async/awaitで受け取り
  ↓ stateを差し替え
トースト通知（成功 or 失敗）
```

## 状態遷移

### メインフロー
初期読み込み（Notion → JSON） → 一覧表示

### 一覧表示からの操作
- **検索中**: filterで絞り込み → 一覧表示
- **タグ検索**: filterで絞り込み → 一覧表示
- **削除確認**: `window.confirm` → 論理削除（`published: false`）→ 一覧表示
- **編集**: `<dialog>`フォーム表示 → タイトル・タグ編集 → 保存 → 一覧表示
- **更新リクエスト**: API POST → 更新中

### 更新フロー（Serverless Function）
更新中（async/awaitで待機） → 更新完了（トースト通知） → state差し替え

### エラー時
更新中 → エラー（トーストでエラー表示） → 一覧表示

## 開発に必要なもの
NotionのAPIキーについては、`.env`ファイルに記載があります。
